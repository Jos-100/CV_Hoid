CURSO DE GIT



What is Git?

Git es un sistema de control de versiones distribuido, lo que significa que hace un clon local del proyecto es un repositorio de control de versiones completo. Estos repositorios locales plenamente funcionales permiten trabajar sin conexión o de forma remota con facilidad. Los desarrolladores confirman su trabajo localmente y, a continuación, sincronizan su copia del repositorio con la copia en el servidor.

En 2002, el proyecto del kernel de Linux empezó a usar un propietario llamado BitKeeper. Instalacion

Descargue e instale Git. Una vez instalado, Git está disponible desde el símbolo del sistema o PowerShell. Se recomienda seleccionar los valores predeterminados durante la instalación. Git no se actualiza automáticamente. Para actualizarlo descargue la nueva versión del instalador, que actualiza Git para Windows y conserva toda la configuración.

COMANDOS PRINCIPALES



GIT INIT

El comando git init crea un nuevo repositorio de Git. Puede utilizarse para convertir un proyecto existente y sin versió en un repositorio de Git, o para inicializar un nuevo repositorio vacío..

GIT STATUS

El comando git status muestra el estado del directorio de trabajo y del área del entorno de ensayo. Permite ver los cambios que se han preparado, los que no y los archivos en los que Git no va a realizar el seguimiento.

GIT BRANCH

El comando git branch te permite crear, enumerar y eliminar ramas, así como cambiar su nombre del repositorio al que pertenecen. Además te permite saber el nombre de la rama en que te encuentras

GIT CHECKOUT

El comando git checkout te permite desplazarte entre las ramas creadas por git branch. Al extraer una rama, se actualizan los archivos en el directorio de trabajo y se indica a Git que registre todas las confirmaciones nuevas en dicha rama.

GIT PUSH

El comando git push se usa para cargar contenido del repositorio local a un repositorio remoto. El envío es la forma de transferir confirmaciones desde tu repositorio local a un repositorio remoto

GIT CLONE

Se utiliza principalmente para apuntar a un repositorio existente y clonar o copiar dicho repositorio en un nuevo directorio, en otra ubicación.

GIT COMMIT

El comando git commit sirve para confirmar una instantánea del directorio del entorno de ensayo en el historial de confirmaciones de los repositorios

GIT PULL

El comando git pull se emplea para extraer y descargar contenido desde un repositorio remoto y actualizar al instante el repositorio local para reflejar ese contenido.
